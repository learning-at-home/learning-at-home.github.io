Why should you care?

Larger neural networks are winning:
  • Pretrained transformers dominate most NLP tasks;
  • Bigger CNNs perform better at computer vision;
  • GPT-3 has 175B parameters and still counting!
With transfer learning, these large models can harness nearly unlimited raw data to improve performance on both academic benchmarks and solve new unexpected tasks.
Image credit: [1] Kaplan et al. (2020), [2, 3] Huang et al. (2018)

That said, training large neural networks isn't cheap. The hardware used for the previous largest language model costs over $25 million. A single training run for GPT-3 will set you back at least $4.6M in cloud GPUs. As a result, researchers can't contribute to state-of-the-art deep learning models and practitioners can't build applications without being supported by a megacorporation. If we want the future of AI to be open, it can't be privately owned.


What is hivemind?


Hivemind is a library for decentralized training of large neural networks. Here's how it works:

To be uploaded...

Here's how a hivemind experiment is organized:
  • Everyone trains one giant model with Mixture-of-Experts layers,
  • Each participant hosts one or more "experts" (aka sub-networks),
  • Peers access each other's experts using a Distributed Hash Table
    - the same type of protocol that powers BitTorrent file sharing.

Training can dynamically scale up as more peers join in. More importantly, hivemind models are fault-tolerant. If some of the k chosen experts fail to process an input, the model can rely on the next best experts available. In other words, Mixture-of-Experts layers handle errors the same way regular neural networks handle dropout.


When should I use it?


Hivemind is designed for three main scenarios:
  • Crowdsourced deep learning using compute from volunteers or decentralized participants;
  • Training neural networks on multiple servers with varying compute, bandwidth and reliability;
  • [TBA] Training huge neural networks across multiple compute clusters.

Conversely, here's what it isn't for:
  • Splitting your model between 2-3 servers that you fully control: use torch.distributed.rpc;
  • Distributed training for a reliable, uniform and highly connected cluster: use DeepSpeed;
  • Training small More specifically, models that fit into a single worker's memory. models with dynamically allocated of in-house workers: use torch.elastic.

Hivemind v0.8 is in the early alpha stage: the core functionality to train decentralized models is there, but the inferface is still in active development. If you want to try hivemind for yourself or contribute to its development, take a look at the quickstart tutorial. Contact us on github with any questions, feedback and issues.
